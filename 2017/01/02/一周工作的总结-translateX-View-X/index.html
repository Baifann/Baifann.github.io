这两个方法都能给控件带来位移效果，但是是略有不同的。
前者是改变控件的translateX属性，而后者是改变x属性。那translateX和x，又有什么不同呢？
translateX代表控件在父类布局中的偏移量，初始化时为0。
而x代表的是控件左上角的坐标（横坐标）。也是相对于父类布局而言的。
在官方文档上有形容x有这么一段说明

This is equivalent to the translationX property plus the current left property.

也就是说x的值是translationX加上left getX() = getTranslationX() + getLeft()
而left暂且不用管。
我们需要知道的是，当一个控件在平移移过程中，控件的left属性是不会改变的，改变的只有x和translateX。由于之前的定义那是不是就意味着，当改变x值时，是不是根本就是改变translateX的值呢？
有关view坐标属性的更多介绍可以参考：http://www.imooc.com/article/3528

影响平移的一些属性

  • 在平移的过程当中，如果需要平移的控件的父布局当中还存在着其他控件，那平移的过程可能不会达到预期。比如会出现更卡顿的情况。
  • 控件所在父布局如果存在padding属性，那也有可能会使预期达到的值不同，比方说translationX属性动画，当父布局如果有paddingLeft = 20属性存在着。调用以下方法。最终控件的绝对位置却到达了320。所以如果可以，尽量保证需要发生的位移的控件处于单独的一层布局中，并且没有margin和padding属性。
